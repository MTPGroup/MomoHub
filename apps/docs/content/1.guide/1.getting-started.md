## Base URL

所有 API 端点的基础 URL 为：

```
https://azusa.hanasaki.tech/v1
```

## 请求格式

- 请求体使用 **JSON** 格式
- 文件上传使用 **multipart/form-data** 格式
- 分页参数通过 **Query String** 传递

## 认证

大部分 API 需要认证。在请求头中携带 Bearer Token：

```bash
curl -X GET https://api.momohub.com/v1/characters \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIs..." \
  -H "Content-Type: application/json"
```

获取 Token 请参阅 [认证指南](/guide/authentication)。

## 分页

列表类接口支持分页查询：

| 参数     | 类型   | 说明                       |
| -------- | ------ | -------------------------- |
| `page`   | number | 页码，从 1 开始            |
| `limit`  | number | 每页数量                   |
| `search` | string | 搜索关键词（部分接口支持） |

分页响应格式：

```json
{
  "items": [...],
  "total": 100,
  "page": 1,
  "limit": 20,
  "totalPages": 5,
  "hasNext": true,
  "hasPrevious": false
}
```

## 通用响应结构

所有接口返回统一的 `ApiResponse<T>` 结构：

```typescript
interface ApiResponse<T> {
  success: boolean
  code: string | null
  message: string
  data: T | null
  errors: Record<string, string> | null
  timestamp: string
}
```

成功响应示例：

```json
{
  "success": true,
  "code": null,
  "message": "OK",
  "data": {
    "id": "char_abc123",
    "name": "助手"
  },
  "errors": null,
  "timestamp": "2025-01-15T08:30:00Z"
}
```
